Branch 1
Vehicle discovery
Session → VDP
Sustain ~48%; lift VDP→ATC indirectly via better intent
1.1
I can't tell from search whether a vehicle has the things I need
Renters can't filter on add-ons, equipment, or rules, so they enter the booking flow just to inspect. This pollutes ATC and burns user effort.
  • → Search filters
1.2
Where vehicles are picked up matters more than I can see in the list
Inspired by Yescapa: maybe pickup location is a stronger filter than we treat it as.
  • → Search map
1.3
The marketplace feels generic, listings don't stand out
Listings blur together; nothing communicates that vehicles are real, distinctive, and currently active.
  • → Vehicle personalisation / name
1.4
No project yet
My request got declined and now I have to start over
After a decline, the email and in-product fallback send the renter into a generic search with no preserved context. Every decline that doesn't loop back is a foregone subsequent FBRS.
1.5
No project yet
I came back to keep looking and lost my place
Returning renters restart from an empty search. Guest state isn't persisted; logged-in users also start over. Likely a meaningful slice of the 169k single-VDP sessions.
1.6
No project yet
I sent an unsuccessful booking request and came back and can't find the other vehicles I liked
The current favorites/wishlist functionality seems hidden and not very accessible. By making that a more prominent part.
1.7
No project yet
I'm planning different trips and I want to save vehicles to different favorite lists to keep my search focused
Vehicles differ in availability and fit for the trip and are found at different locations, so different trips require different favorite lists.
Branch 2
Vehicle selection
VDP → ATC
+1 to 2pp absolute (5% to 6 or 7%)
VDP inspiration: QuirkyCampers
2.1
I don't know what booking actually involves before I commit
First-timers convert at 17% vs 24% overall. The VDP sells the vehicle but says nothing about the P2P process.
  • → Timeline for booking on VDP
2.2
I can't tell if other people trust this owner / vehicle
Reviews are flat (single star, no context). Renters lack social proof to reduce hesitation.
  • → Rich reviews
  • → Social proof
2.3
I can't see total price on mobile without entering the booking flow
Mobile users can't see cost overview unless they ATC. The button asks too much for too little visible value.
  • → Unify ATC button on desktop and mobile
2.4
Biggest lever
This vehicle isn't quite it, and I don't see anywhere to go from here
~102k sessions per 2 months land on a vehicle that doesn't match, then bounce. Turn the VDP into a path to a fitting vehicle without making the user start over. Sized at 50 to 140% of the KR gap on its own.
  • → Vehicle recommendations on VDP
  • → Arrow navigation to next VDP
2.5
No project yet

2.6
No project yet
I want to compare vehicles
Overlap with favorites/wishlist (Opp 1.6); comparison may live as a feature on top of a more accessible favorites surface.
2.7
No project yet

The middle 'Listing details' block pushes trust signals and CTA way down. Tabs deliver scan + drill while keeping all content server-rendered for SEO.
Branch 3
Booking flow
ATC → FBRS
First-timer 17% to ~18%; overall 25% to ~26%
3a Empty / low-value steps
3a.1
I'm being asked things that don't apply to me
Customize and Insurance steps render even when there's nothing to choose.
  • → Skip addons and insurance when possible
  • → Buy addons after booking
3a.2
The first step of the flow kills me before I've engaged
27% drop on the first step. Hypothesis: it's the position, not the content.
  • → A/B test: insurance step before addons
3b Sign-up / login (worst single dropoff)
3b.1
Login feels like a separate task that interrupted my booking
Step converts at 68% globally; DACH+INT 65% vs Nordics 78%.
  • → Sign-up/login as overlay instead of inline step
  • → Offer registration with Phone number
3c About-you (Nordic black hole)
3c.1
Bank ID before I even know if the owner accepts feels excessive
Nordic eID is a pre-booking gate. 44% of SE about-you sessions never start eID. Once started, ~89% complete; the block is cognitive, not technical.
  • → Move Nordic eID verification to after booking like Onfido
3c.2
I'm filling out the same person twice (or for the wrong person)
Renters are confused by main-driver vs account-holder, and re-fill data on every booking attempt.
  • → Improve about-you step
  • → Separate between main driver and account on about-you
  • → Main driver details and additional driver
  • → Save DL to profile
  • → Make the Driver's license question clearer
3c.3
My country isn't in the list
Renters from 12 EU countries can't select their country of residence. ~50% drop on About-you for international segment.
  • → Allowlist all EU countries for renters
3d Payment
3d.1
I can't see all my payment options
'Pay later' is below the fold. Many users don't notice the option that would have unblocked them.
  • → Show all payment options above the fold
3d.2
Payment fails and I don't know why
3D Secure, payment limits, online-purchase locks; we can't know which, but we can prime users.
  • → Payment step, many users struggle with the payment
3d.3
The cancellation policy makes me anxious right at commit
Showing a near-closed free-cancellation window creates last-second anxiety. Best treated upstream via 2.1. The BDP timeline below is post-booking; impact on this initiative's KR is indirect.
  • → Timeline for booking in BDP
3e Catch-alls and infrastructure
3e.1
Small fixes and foundation
Bucket for small fixes (mandatory-field markers, error displays) plus the foundation for clean A/B reads on every project.
  • → Misc Booking flow improvements
  • → Feature flags
Strategic questions, not yet projects
What should 'Buchungsanfrage' actually be?
Binding booking framed as a 'request' is a likely root cause of the 17% first-timer CR. Touches pricing, ops load, owner experience, competitive positioning.
What does the first-timer booking experience need to look like?
First-timer 17% vs 24% overall, but funnel events don't distinguish first-timers per step. Investing in event tracking might be the prerequisite to every other first-timer experiment.
Explicitly killed / out of scope
Move 'message owner' button next to ATC on VDP
Rejected. Would push us toward a classified model; we want trust through supply quality + instant-book, not message-first.
BDP additional driver display confusion
Post-booking only; users already successful.
